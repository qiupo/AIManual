# 提示词入门

提示词（Prompt）是你与AI沟通的桥梁。学会编写有效的提示词，是掌握AI工具的核心技能。

## 什么是提示词？

**提示词**就是你对AI说的话、提出的问题或给出的指令。

```
┌─────────┐                    ┌─────────┐
│  你的   │  → 提示词 →         │   AI    │
│  想法   │                    │ 的回应  │
└─────────┘                    └─────────┘

例子：
提示词："解释什么是递归"
回应：递归是...
```

### 为什么提示词很重要？

同样的任务，不同的提示词会得到完全不同的结果：

❌ **提示词1**："写个爬虫"

✅ **提示词2**："用Python写一个网页爬虫，
     爬取某新闻网站的标题和链接，
     使用requests和BeautifulSoup库，
     添加异常处理和延时功能，
     代码要有详细注释"

**结果差异**：提示词1可能得到一个简单但不实用的代码，
           提示词2则得到一个完整可用的解决方案。

## 提示词的黄金结构

一个优秀的提示词通常包含以下几个部分：

### 结构模板

```
[角色设定] + [任务描述] + [具体要求] + [输出格式] + [上下文]
```

### 示例拆解

```
完整提示词：
"你是一位经验丰富的Python开发者[角色]。
请帮我写一个数据清洗脚本[任务]。
要求：处理缺失值、去重、格式转换[具体要求]，
使用Pandas库，代码要有详细注释[技术要求]。
输出格式：Python代码 + 使用说明[输出格式]。
数据集是CSV文件，包含100万条用户记录[上下文]。"
```

## 提示词的核心要素

### 要素1：角色设定（Role）

告诉AI以什么身份回答问题：

```
❌ 无角色："如何选择数据库？"

✅ 有角色："你是一位系统架构师，
          有10年大型项目经验。
          如何为一个高并发电商网站
          选择合适的数据库？"
```

**常用角色**：
- 教师/讲师（教学和解释）
- 专家/顾问（专业建议）
- 编辑/作家（内容创作）
- 程序员（代码开发）
- 产品经理（产品思考）
- 面试官（模拟面试）

### 要素2：任务描述（Task）

清晰说明你想要AI做什么：

```
❌ 模糊："帮我优化这个"

✅ 清晰："帮我优化这段Python代码的执行效率。
         目前处理10万条数据需要5分钟，
         希望能优化到30秒以内。"
```

**任务动词**：
- 解释、分析、比较
- 编写、创建、生成
- 优化、改进、重构
- 总结、提取、简化
- 评估、判断、推荐

### 要素3：具体要求（Requirements）

详细说明完成任务的标准：

```
✅ 好的要求：
- 字数：500-800字
- 语言：通俗易懂，适合初学者
- 结构：引言-主体-结论
- 包含：3个实际案例
- 避免：过于专业的术语
- 语气：友好、鼓励性
```

**要求维度**：
- **长度**：字数、段落数量
- **难度**：入门、中级、高级
- **结构**：章节、层次
- **内容**：包含/避免的要素
- **风格**：正式/轻松、专业/通俗
- **格式**：列表、表格、代码块

### 要素4：输出格式（Format）

指定AI回答的格式：

```
✅ 格式示例：
- 用表格对比
- 分步骤列出
- Markdown格式
- JSON格式
- 代码块形式
- 思维导图结构
```

**示例**：

```
"请用以下格式回答：
## 优点
1. ...
2. ...

## 缺点
1. ...
2. ...

## 适用场景
- ...
- ..."
```

### 要素5：上下文信息（Context）

提供背景信息，帮助AI更好地理解：

```
✅ 上下文包含：
- 目标：我正在学习Web开发
- 背景：已经掌握HTML/CSS，开始学JavaScript
- 困难：对异步编程理解困难
- 期望：用生活中的例子解释
```

## 提示词的6个层次

让我们通过6个层次看提示词的进化：

### Level 1：基本提问

```
"什么是机器学习？"
```

**特点**：简单直接，适合基础问题

### Level 2：添加要求

```
"解释什么是机器学习，
用通俗易懂的语言，
适合完全不懂技术的人。"
```

**特点**：增加了目标受众和语言要求

### Level 3：提供结构

```
"解释什么是机器学习：
1. 用简单定义开始
2. 给一个生活例子
3. 说明主要应用
4. 总结其重要性
用非技术语言。"
```

**特点**：明确回答的结构和步骤

### Level 4：添加角色和场景

```
"你是一位大学计算机老师，
正在给非专业学生上通识课。
请解释什么是机器学习，
用生活中的类比，
避免数学公式，
最后布置一个思考题。"
```

**特点**：设定角色和场景，更符合实际需求

### Level 5：详细约束和反馈

```
"你是一位资深AI教育专家。
准备一份30分钟的演讲稿，
主题：'给中学生的AI入门'。

要求：
- 开场要吸引人
- 包含2-3个互动问题
- 用3个实际案例
- 避免技术术语
- 鼓励性结尾
- 字数：1500字左右

完成后，给出一个演讲要点清单。"
```

**特点**：详细的约束条件和后续要求

### Level 6：多阶段任务

```
"阶段1：先给我这个演讲的大纲
阶段2：确认大纲后，我再说如何细化
阶段3：最后帮我准备PPT的结构

让我们从阶段1开始：演讲大纲"
```

**特点**：分阶段执行，每阶段确认后再继续

## 常见提示词模板

### 模板1：学习新概念

```
"你是一位[领域]专家/老师。
请解释[概念]，要求：
- 用[具体方式]的比喻
- 适合[目标读者]水平
- 包含[数字]个实际例子
- 避免[不要什么]
- 最后给出[行动建议]"
```

**使用示例**：

```
"你是一位数据库专家老师。
请解释'数据库索引'这个概念，要求：
- 用图书馆索引做比喻
- 适合编程初学者
- 包含3个实际应用场景
- 避免复杂的数据结构术语
- 最后给出何时应该创建索引的建议"
```

### 模板2：代码编写

```
"我需要[编程语言]代码实现[功能描述]。

上下文：
- 使用场景：[场景]
- 输入：[输入说明]
- 期望输出：[输出说明]
- 特殊要求：[性能/安全等]

技术要求：
- 使用[库/框架]
- 添加[异常处理/测试等]
- 代码要有详细注释
- 遵循[编码规范]

最后说明：
1. 代码的工作原理
2. 可能的问题和注意事项
3. 如何测试验证"
```

### 模板3：内容创作

```
"你是一位[角色，如：专业编辑/作家]。
帮我创作[类型，如：文章/邮件/报告]。

主题：[主题]
目标读者：[读者群]
目的：[要达到的效果]

结构要求：
[列出具体结构]

内容要求：
- 字数：[范围]
- 语气：[正式/轻松等]
- 包含：[必须包含的要素]
- 避免：[不要的内容]

格式：[Markdown/HTML等]

完成后，给出：改进建议/替代版本"
```

### 模板4：问题诊断

```
"我遇到了[问题类型]问题。

问题描述：
[详细描述问题]

环境信息：
- 系统：[操作系统等]
- 软件/语言版本：[版本号]
- 相关配置：[配置信息]

已尝试：
1. [尝试1]：结果
2. [尝试2]：结果

错误信息：
[粘贴错误信息]

期望：
- 系统分析可能的原因
- 给出排查步骤（从最可能到最不可能）
- 提供解决方案
- 说明如何预防类似问题"
```

### 模板5：对比分析

```
"请对比[A]和[B]。

对比维度：
1. [维度1]
2. [维度2]
3. [维度3]

使用场景：
- 我的需求是：[具体需求]
- 我的背景是：[个人情况]
- 我的限制是：[预算/时间等]

请：
- 用表格形式对比
- 针对我的情况给出建议
- 说明选择理由
- 提及可能的替代方案"
```

### 模板6：学习路径

```
"我想学习[技能/领域]。

我的情况：
- 当前水平：[完全零基础/有一定基础等]
- 可用时间：[每天/每周多少时间]
- 学习目标：[具体目标]
- 时间期限：[希望多久学会]

请提供：
1. 分阶段的学习计划（每个阶段的目标和内容）
2. 推荐的学习资源（免费/付费）
3. 实战项目建议
4. 常见误区和避坑指南
5. 如何验证学习效果

要求：循序渐进，每步都可执行"
```

## 提示词优化技巧

### 技巧1：示范和举例

```
"参考下面这个例子的风格：
[示例]

用类似的方式，写一个关于[新主题]的介绍"
```

### 技巧2：思维链引导

```
"让我们一步步思考：
1. 首先，分析...
2. 然后，考虑...
3. 最后，得出..."

"请展示你的思考过程"
```

### 技巧3：要求反思和改进

```
"在你给出答案后，请：
1. 自我评估这个答案的完整性
2. 指出可能的不足
3. 提供改进版本"
```

### 技巧4：设定约束条件

```
"在回答时，请遵守以下约束：
- 每个观点不超过50字
- 必须包含数据支撑
- 避免使用被动语态
- 使用列表而非段落"
```

### 技巧5：提供反馈循环

```
"先给我一个初步方案。
我会给出反馈，
然后你再根据反馈优化。
我们先从初步方案开始。"
```

## 不同场景的提示词示例

### 场景1：理解复杂概念

```
"用'教5岁孩子'的方式解释'区块链'。
要求：
1. 使用糖果或玩具做类比
2. 不超过200字
3. 包含'为什么需要区块链'的说明"
```

### 场景2：代码调试

```
"这段代码报错了：
[代码]

错误信息：
[错误信息]

请：
1. 解释错误的根本原因
2. 说明为什么会发生这种错误
3. 提供修复方案
4. 给出预防建议"
```

### 场景3：创意写作

```
"为一款智能家居产品写产品介绍。

产品特点：
- 语音控制
- 远程操作
- 节能环保

目标：让年轻人觉得这很酷
语气：时尚、简洁、有科技感
字数：300字以内
口号要朗朗上口"
```

### 场景4：决策辅助

```
"我在两个工作offer之间犹豫：

Offer A：
- 大厂，薪资高
- 但工作压力大
- 可能需要加班

Offer B：
- 创业公司，薪资一般
- 但成长空间大
- 工作生活平衡

作为职业规划顾问，
你会如何分析？请用决策矩阵方法。"
```

### 场景5：技能提升

```
"我已经会Python基础语法，
想进阶到高级开发。

请帮我：
1. 评估当前水平（我会：list, dict, 函数, 类）
2. 列出需要学习的高级主题
3. 为每个主题推荐学习资源
4. 设计3个递进式项目
5. 给出学习时间表（每周10小时）"
```

## 提示词自查清单

在发送提示词前，检查：

- [ ] **目标明确**：AI知道我要什么吗？
- [ ] **上下文充分**：提供了足够的背景信息吗？
- [ ] **要求具体**：说明了标准和约束吗？
- [ ] **格式清晰**：指定了输出格式吗？
- [ ] **语言精确**：避免了模糊和歧义吗？
- [ ] **角色设定**：需要设定特定角色吗？
- [ ] **长度适中**：不太简单也不过于复杂？

## 常见提示词错误

### ❌ 错误1：过于宽泛

```
"讲讲人工智能"
```

**✅ 改进**：

```
"用30分钟的时间，
为文科生介绍人工智能的发展历史，
重点讲解3个里程碑事件，
用故事化的方式。"
```

### ❌ 错误2：缺少上下文

```
"这段代码有错吗？"
```

**✅ 改进**：

```
"我写的这段代码应该读取文件并计算平均值，
但运行结果是0，找不到问题。
代码如下：
[代码]
文件内容格式是：
[示例]
请帮我找出错误。"
```

### ❌ 错误3：相互冲突的要求

```
"写一个技术文档，
既要专业详尽，
又要通俗易懂，
还要控制在500字以内。"
```

**✅ 改进**：

```
"写一个技术文档概述，
控制在500字以内。
然后为每个部分提供
详细版本的可选链接。"
```

## 🎯 实战练习

### 练习1：提升提示词

```
原始提示词："写个爬虫"

任务：将其升级到Level 5，
       添加角色、要求、格式等要素
```

### 练习2：对比实验

```
任务：用3个不同详细程度的提示词，
      让AI完成同一任务，
      观察结果差异

任务主题自选，比如：
- 解释一个概念
- 写一段代码
- 创作一个故事
```

```
任务：选择本章节的一个模板，
      填入你自己的内容，
      实际使用并记录效果
```

## 小结

- **提示词是AI沟通的核心技能**
- **结构化的提示词**包含角色、任务、要求、格式、上下文
- 提示词可以**分6个层次**逐步完善
- 掌握**常用模板**能快速构建有效提示词
- **实践和迭代**是提升提示词水平的唯一途径
